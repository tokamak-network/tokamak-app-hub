# 구현 로드맵
## Tokamak App Hub - Implementation Roadmap

## 1. 타임라인 개요

```
Week 1 (Day 1-5)
├── Wave 1: 인프라 & 기반 (병렬 실행)
│   ├── [Task 1] 프로젝트 초기 설정 ────────────────────────── 1-2일
│   ├── [Task 2] 데이터 모델 및 스키마 정의 ──────────────────── 1일
│   └── [Task 3] UI 디자인 시스템 구축 ────────────────────── 1-2일
│
│   마일스톤 1: 기본 프로젝트 구조 완성
│   - Next.js 15 프로젝트 실행 가능
│   - Tokamak 브랜딩 적용된 UI 컴포넌트
│   - 샘플 앱 데이터 3개 이상

Week 2 (Day 6-10)
├── Wave 2: 핵심 기능 (병렬 실행)
│   ├── [Task 4] 앱 목록 페이지 구현 ────────────────────────── 2일
│   ├── [Task 5] 앱 상세 페이지 구현 ────────────────────────── 2일
│   └── [Task 6] 검색 및 필터 기능 구현 ─────────────────────── 2일
│
│   마일스톤 2: 핵심 사용자 경험 완성
│   - 앱 목록 카드 그리드 동작
│   - 앱 상세 페이지 GitHub 통계 표시
│   - 검색 및 필터 URL 동기화 동작

Week 3 (Day 11-14)
├── Wave 3: 등록 & 마무리 (순차 실행)
│   ├── [Task 7] 앱 등록 페이지 구현 ────────────────────────── 1-2일
│   ├── [Task 8] GitHub API 연동 최적화 ──────────────────────── 1일
│   └── [Task 9] 최종 통합 및 배포 설정 ──────────────────────── 1일
│
│   마일스톤 3: MVP 완성
│   - 앱 등록 폼 → GitHub Issue 연동 동작
│   - GitHub API ISR 캐싱 적용
│   - Vercel 배포 가능 상태
│   - Lighthouse 80점 이상

└── 출시 준비 완료 (Day 14)
```

## 4. 마일스톤 상세

### 마일스톤 1: 기본 프로젝트 구조 (Day 3)

**완료 조건**:
- [x] Next.js 15 프로젝트 생성 및 실행 가능
- [x] shadcn/ui 설치 및 초기화
- [x] Tokamak 브랜드 컬러 적용
- [x] 기본 레이아웃 (Header, Footer) 구현
- [x] TypeScript 타입 및 Zod 스키마 정의
- [x] 샘플 앱 데이터 3개 이상

**검증 방법**:
```bash
bun run dev    # 개발 서버 실행
bun run build  # 빌드 성공
bun run lint   # 린트 통과
```

### 마일스톤 2: 핵심 사용자 경험 (Day 7)

**완료 조건**:
- [x] 메인 페이지 앱 카드 그리드 표시
- [x] 앱 상세 페이지 GitHub 통계 표시
- [x] 검색 기능 동작 (디바운스)
- [x] 카테고리 필터 동작
- [x] URL 파라미터 동기화

**검증 방법**:
- Playwright 테스트 또는 수동 검증
- URL 파라미터 확인: `?q=test&category=sdk`

### 마일스톤 3: MVP 완성 (Day 10-14)

**완료 조건**:
- [x] 앱 등록 폼 → GitHub Issue 리다이렉트 동작
- [x] GitHub API ISR 캐싱 적용
- [x] Vercel 배포 완료
- [x] Lighthouse 점수 80점 이상
- [x] 반응형 디자인 검증

**검증 방법**:
```bash
# Lighthouse 검사
bunx lighthouse https://tokamak-app-hub.vercel.app --view
```
