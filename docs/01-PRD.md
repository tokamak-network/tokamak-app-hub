# PRD (Product Requirements Document)
## Tokamak App Hub - 제품 요구사항 문서

---

## 1. 문제 정의

### 배경
- Tokamak Network에서 AI를 활용해 월 수십 개의 블록체인 앱/패키지가 생성됨
- 현재 이들을 한 곳에서 모아 볼 수 있는 중앙화된 카탈로그가 없음
- 사내 개발자들이 생성된 앱을 발견하고 재사용하기 어려움

### 문제
- 앱/패키지가 여러 GitHub 저장소에 분산되어 있어 검색과 발견이 어려움
- 새로 생성된 앱에 대한 가시성 부족
- 중복 개발 가능성 증가

---

## 2. 타겟 사용자 및 페르소나

| 페르소나 | 설명 | 주요 니즈 |
|----------|------|----------|
| **내부 개발자** | Tokamak Network 소속 개발자 | 기존 앱/패키지 검색, 재사용, 새 앱 등록 |
| **AI 시스템** | 앱 자동 생성 시스템 | 생성된 앱을 마켓플레이스에 등록 |
| **프로젝트 매니저** | 팀 리드, PM | 생태계 현황 파악, 앱 카탈로그 관리 |
| **외부 방문자** | 오픈 소스 커뮤니티 | Tokamak 생태계 탐색, 앱 발견 |

---

## 3. 핵심 기능 (MVP)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **앱 목록 조회** | 카드 그리드 형태로 모든 앱 표시 | P0 |
| **검색** | 이름, 설명, 태그 기반 실시간 검색 | P0 |
| **카테고리 필터** | dApp, SDK, Tool 등 카테고리별 필터링 | P0 |
| **태그 필터** | 자유 태그 기반 필터링 | P1 |
| **앱 상세 페이지** | GitHub 통계, 설명, 링크 표시 | P0 |
| **앱 등록** | GitHub URL 입력으로 새 앱 제출 | P0 |
| **반응형 디자인** | 모바일/태블릿/데스크톱 지원 | P0 |

---

## 4. 향후 기능 (Post-MVP)

| 기능 | 설명 | 예상 시기 |
|------|------|----------|
| 다크 모드 | 다크/라이트 테마 전환 | Phase 2 |
| 자동 앱 수집 | 특정 토픽/태그 저장소 자동 등록 | Phase 2 |
| 사용자 인증 | GitHub OAuth 기반 로그인 | Phase 2 |
| 댓글/리뷰 | 앱에 대한 피드백 시스템 | Phase 3 |
| 인기도 순위 | 조회수, 다운로드 기반 랭킹 | Phase 3 |
| README 렌더링 | 상세 페이지에서 README 표시 | Phase 2 |
| 자동 PR 생성 | 등록 폼 → 자동 PR 생성 | Phase 2 |

---

## 5. 성공 지표 (KPIs)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 등록된 앱 수 | MVP 출시 시 20개 이상 | JSON 파일 카운트 |
| 월간 신규 등록 | 10개 이상 | GitHub Issue 수 |
| 페이지 로드 시간 | < 2초 | Lighthouse |
| Lighthouse 점수 | 80점 이상 | Chrome DevTools |
| 검색 정확도 | 관련 결과 상위 5개 내 | 수동 테스트 |

---

## 6. 제약 조건

### 기술적 제약
- 데이터베이스 없이 JSON 파일 기반 저장
- GitHub API Rate Limit (인증 토큰 사용 시 5000 req/hour)
- 정적 호스팅 (Vercel) 기반

### 비기술적 제약
- 공개 접근 (인증 없음)
- 누구나 등록 가능 (관리자 승인 필요)

---

## 7. 릴리즈 기준

### MVP 릴리즈 체크리스트
- [ ] 앱 목록 페이지 동작
- [ ] 앱 상세 페이지 동작 (GitHub 통계 표시)
- [ ] 검색 기능 동작
- [ ] 카테고리 필터 동작
- [ ] 앱 등록 폼 → GitHub Issue 연동
- [ ] 반응형 디자인 (모바일/데스크톱)
- [ ] Lighthouse 점수 80점 이상
- [ ] 샘플 앱 데이터 10개 이상
